Réalisation du projet

  • Analyse des besoins

    • Dans le cadre de notre projet de mentorat notre application doit etre dans la capaciter de permettre l'inscription et la connection des utilisateur puis leur attribuer en fonction de leurs capacité un profil,de planifier les séances de travail et de stoqué les messages échangés.Il doit etre aussi en mesure de sécuriser les données personnelles des utilisateurs,etre rapide dans son fonctionnement et offrir une bonne experiance aux utilisateurs.

  • conception

    • Notre projet a été conçue sur la base d'un frontend,un backend et une base de données. L'application adopte une architecture client-serveur modulaire, avec une séparation claire entre les couches de présentation, de logique métier et d'accès aux données.

  • Développement

    • Le développement de notre projet vise à concevoir une plateform web permettant de mettre en relation diférent étudiant sous les profil de mentor et mentoré dans différente matière.Cette plateforme offre les fonctionnalitées telles que l'inscription et la connection des utilisateurs,la gestion des profils,l'attribution des mentors ainsi que la planification des séances.
    Le frontend a été réalisé avec HTML,CSS et JavaScrit dans le but d'obtenir une interface simple mais fonctionnelle. Le backend a été déveveloppé avec python et java script afin de pourvoir derer les données des utilisateurs ainsi que les interration entrer utilisateur et la base de données avec postgrèSQL pour stocké les différente table(information)

Difficultés rencontrées et Solution apportées

  • Configuration de la base de données pour les tests
    • Les 30 tests échouaient avec SQLALCHEMY_DATABASE_URI must be set. Résolution : ajouter load_dotenv() dans conftest.py pour que le .env soit chargé avant les tests.
  • Compatibilité PowerShell vs Bash
    • Les commandes Linux comme mkdir -p app/models app/routes ne fonctionnaient pas sur Windows PowerShell. Résolution : exécuter les mkdir un par un.
  • Conflit Git au premier push
    • Le dépôt GitHub contenait déjà des fichiers, ce qui rejetait le push. Résolution : git pull origin main --allow-unrelated-histories avant de pusher.
  • Éditeur Vim inattendu lors du merge
    • Git a ouvert Vim pour le message de merge, bloquant le terminal. Résolution : taper :wq puis Entrée pour sauvegarder et quitter.
  • Mauvaise version de flask-sqlalchemy
    • La version installée (0.6) était incompatible avec la syntaxe moderne. Détecté grâce à l'avertissement de PyCharm sur requirements.txt.
  • Gestion de la messagerie

Guide d'utilisation

Guide dutilisation pour les utilisateurs finaux de mentor_link.Il décrit les différentes fonctionnalitées de la platforme

1. Inscription

Pour créer un compte, l'utilisateur accède à la page d'inscription, renseigne ses informations personnelles (nom, prénom, email, téléphone et mot de passe), puis valide le formulaire.

2. Connexion

Après son inscription, l'utilisateur peut se connecter à l'aide de son adresse email ou de son numéro de téléphone ainsi que de son mot de passe.

3. Gestion du profil

L'utilisateur peut compléter ou modifier son profil en ajoutant :

  • Sa filière d'étude ;
  • Son niveau académique ;
  • Ses compétences ;
  • Ses lacunes ;
  • Ses disponibilités ;
  • Une photo de profil .
4. Publication d'une offre ou d'une demande

Les étudiants peuvent publier une offre de mentorat lorsqu'ils souhaitent aider d'autres étudiants ou une demande lorsqu'ils recherchent un accompagnement.

5. Consultation des suggestions

Le système analyse les profils et propose automatiquement des correspondances en fonction des compétences, des besoins et des disponibilités des utilisateurs.

6. Messagerie

Une fois une correspondance établie, les utilisateurs peuvent échanger des messages en temps réel grâce au module de messagerie intégré à la plateforme.

7. Notifications

Les utilisateurs reçoivent des notifications lorsqu'ils obtiennent un nouveau match, un nouveau message ou lorsqu'une demande est acceptée.

8. Déconnexion

L'utilisateur peut quitter sa session à tout moment en cliquant sur le bouton « Déconnexion » disponible dans le menu principal.
